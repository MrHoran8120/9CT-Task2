Good Coding Practice

Writing Clean, Understandable Code

Good coding practice means writing code that is clear, organised, and easy for others (and your future self) to understand. Well-structured code helps prevent errors, simplifies debugging, and makes collaboration easier. The following guidelines describe how to write effective and maintainable Python code for this assessment task.

1. Use Functions to Organise Code

Code should be divided into logical chunks called functions. Each function should handle one specific task or process, making your program modular and easier to test. For example: 

# Function to display a welcome message
def display_welcome():
    print("Welcome to the Python Menu Program!")

2. Single Program File Structure

For this project, all code for your menu system must be contained in a single program file named:

9999999-Menu.py

Replace 9999999 with your student ID. You must not import or execute any external Python scripts or use third-party libraries. Your program should run entirely on its own, using only standard Python commands.

3. Comment Your Code Clearly

Comments explain what your code does and why it exists. They are essential for both collaboration and marking.

  • Start each program with a brief comment describing its overall purpose.
  • Each function should begin with a short description of what it does and any important inputs or outputs.
# Program: 9999999-Menu.py
# Purpose: Main menu program for the Python assessment task

def display_menu():
    # Function: display_menu
    # Purpose: Show all available options to the user
    print("1. Hello World")
    print("2. Calculator")
    print("3. Encryption Tool")
    print("4. Exit")

4. Naming Conventions

  • Use snake_case for variable and function names (e.g., total_score, calculate_average()).
  • Use descriptive names that clearly show what the variable or function represents.
  • Avoid single-letter variable names except in short loops (e.g., for i in range(5):).

5. Keep Output Clean and Consistent

All output should be formatted neatly and consistently, matching the required model output exactly. Watch spacing, punctuation, and capitalisation carefully. The assessorâ€™s comparison tool will expect an identical match.

6. Test Frequently

Run your code regularly as you build it. Test each function separately before integrating it into your menu program. Regular testing helps catch logical and syntax errors early.

7. Readability Over Complexity

Clear, readable code is always better than clever but confusing code. Use spacing and indentation properly, and make your logic easy to follow. Future programmers (including you) should be able to read your work and immediately understand what it does.
