Terminal Sessions and Menu Programs

What Is a Terminal Session?

A terminal session is a text-based environment where users interact with the computer entirely through the keyboard, not the mouse. It uses a command-line interface (CLI) rather than windows, buttons, or icons. Users type commands and press Enter to execute them, and the computer responds with text-based output.

Terminals are used to run programs, navigate folders, manage files, and perform system tasks efficiently. Common examples include the Windows Command Prompt, PowerShell, macOS Terminal, and Linux shells such as Bash or Zsh. In programming, terminal sessions are essential for running scripts, testing code, and working with development tools.

What Is a Menu Program?

A menu program is a simple text-based application that runs inside a terminal. It displays a numbered list of options and waits for the user to type a choice using the keyboard. The program then performs an action based on the selection—such as displaying data, calculating results, or opening another menu.

Because there is no mouse interaction, terminal menu programs teach students to design programs that are entirely controlled by text input. This strengthens understanding of loops, conditionals, and functions—the core building blocks of structured programming.

Why Learn Menu Programs?

  • They teach students to create programs that accept typed input instead of using a mouse.
  • They demonstrate how users interact with console-based software.
  • They combine input, processing, and output in a repeating control loop.
  • They build confidence in logic, sequencing, and user feedback design.
  • They promote good commenting and code readability habits.

Learning to build and navigate menu programs in the terminal helps students understand how computers process instructions. This skill forms the foundation for developing larger applications and graphical user interfaces in the future.
